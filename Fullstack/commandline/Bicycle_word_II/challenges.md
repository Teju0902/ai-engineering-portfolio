# Challenges and Solutions

## Challenge 1: Print the Working Directory

**Task:** Display the current working directory.

**Solution:**
Used the `pwd` command to identify the current location in the filesystem.

## Challenge 2: List Files and Directories

**Task:** View the contents of the working directory.

**Solution:**
Used the `ls` command to display available files and folders.

## Challenge 3: Navigate to the Freight Directory

**Task:** Move into the `freight/` directory.

**Solution:**
Used `cd freight` to change directories.

## Challenge 4: Explore the Freight Directory

**Task:** View contents inside `freight/`.

**Solution:**
Used `ls` to inspect available subdirectories.

## Challenge 5: Navigate to the Porteur Directory

**Task:** Move into the `porteur/` directory.

**Solution:**
Used `cd porteur`.

## Challenge 6: Return to the Root Directory

**Task:** Move up two directory levels and verify contents.

**Solution:**
Used `cd ../..` followed by `ls`.

## Challenge 7: Navigate to Mountain/Downhill

**Task:** Reach the `mountain/downhill/` directory.

**Solution:**
Used `cd mountain/downhill`.

## Challenge 8: Create dirt.txt

**Task:** Create a new file named `dirt.txt`.

**Solution:**
Used `touch dirt.txt`.

## Challenge 9: Create mud.txt

**Task:** Create another file named `mud.txt`.

**Solution:**
Used `touch mud.txt`.

## Challenge 10: Verify New Files

**Task:** Confirm the files were created successfully.

**Solution:**
Used `ls` to display directory contents.

## Challenge 11: Create Safety Directory

**Task:** Add a new directory named `safety/`.

**Solution:**
Used `mkdir safety`.

## Challenge 12: Return to bicycle-world-ii

**Task:** Navigate back to the project root.

**Solution:**
Used `cd ../..`.

## Challenge 13: Inspect Project Structure

**Task:** View all files and folders in the root directory.

**Solution:**
Used `ls`.

## Challenge 14: Add a BMX Category

**Task:** Create a new directory called `bmx/`.

**Solution:**
Used `mkdir bmx`.

## Challenge 15: Create tricks.txt in BMX

**Task:** Create a file named `tricks.txt` inside `bmx/` without entering the directory.

**Solution:**
Used `touch bmx/tricks.txt`.

## Challenge 16: Verify Final Filesystem

**Task:** Display the final contents of the current directory.

**Solution:**
Used `ls` to confirm all newly created files and directories were present.

## Summary

This project provided hands-on experience with fundamental Linux command-line operations, including:

* Filesystem navigation using `cd`
* Viewing directory contents using `ls`
* Checking location with `pwd`
* Creating files using `touch`
* Creating directories using `mkdir`
* Working with relative paths
* Managing hierarchical directory structures

By completing all 16 tasks, I gained practical experience with essential terminal commands commonly used by developers and system administrators.
